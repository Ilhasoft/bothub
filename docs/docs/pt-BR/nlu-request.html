NLU Request

Você pode fazer requisições HTTP para os pontos de acesso do NLU disponíveis no serviço HTTP do NLP.

Os pontos de acesso disponíveis são:

CaminhoDescriçãoDocumentação
/infoObter informações do repositórioAcessar documentação
/parseAnalisar textoAcessar documentação
/trainTreinar nova versão do botAcessar documentação

Credenciais

As credenciais devem ser passadas pelo cabeçalho Authorization em cada requisição. Certifique-se que você tenha no cabeçalho Authentication: Bearer [Seu Authorization Token] em todas as requisições HTTP.

Authorization Token

Para fazer requisições você precisa de um Authorization Token. Ele identifica o usuário e qual bot (repositório) é responsável para processar a requisição, em outras palavras, cada usuário tem um token para casa repositório que ele tem acesso.

Você pode recuperar este Authorization Token visitando a página do seu repositório no Bothub Webapp na aba Analizar ou fazer uma requisição autenticada para API no ponto de acesso Authorization Token.

O Authorization Token é um UUID utilizando a representação canônica textual, a seguir:

Ex:

123e4567-e89b-12d3-a456-426655440000
xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
